# Day 1 – Environment Setup and Jupyter Workflow

## Objective

The goal of Day 1 was to prepare the Python environment and project
structure that will be used throughout the AI and Machine Learning
internship.

## Work Completed

During this day, I:

- Created the internship Git repository.
- Connected the local repository to GitHub.
- Created the `week-1` folder.
- Created a Python virtual environment named `.venv`.
- Activated the virtual environment using PowerShell.
- Installed Jupyter Notebook and the required data science libraries.
- Practiced using code cells and Markdown cells in Jupyter.
- Created a `requirements.txt` file to record the installed packages.
- Added a `.gitignore` file to exclude the virtual environment and
  temporary Python files.
- Committed and pushed the completed setup work to GitHub.

## Files

- `01_environment_setup.ipynb` – demonstrates the environment setup and
  confirms that the main libraries are installed.
- `../requirements.txt` – contains the Python packages used during Week 1.

## Starting the Environment

From the `week-1` folder, the existing virtual environment can be
activated using:

```powershell
.\.venv\Scripts\Activate.ps1
```

Jupyter Notebook can then be started using:

```powershell
jupyter notebook
```

## What I Learned

I learned why a virtual environment is useful, how Jupyter runs as a
local server, and how Git and GitHub are used to track and store the
internship work.
